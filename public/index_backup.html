Input

Input all the required files using a form

These includes Rc,Dl,Pancard,Policy,Report,Vehicle Image,Mapping file and Authoritative Data

All these files are scanned using google vision api which return all the words present in the files

Google vision api result (OCR).

All the words present in the files are extracted and are saved in JSON format

Now the next step is to convert the unstructured data to meaningful data using mapping file

Structured Output

On applying OCR we get a bag of words of the processed document .Since the physical claim files is not in tabular form hence on scanning it is not necessary that our required information will occur sequentially . For eg : To extract DL no. from driver license ,we first generate a bag of all words scanned from DL.Then we search for “DL number” keyword in the bag of words and store all the numbers near this keyword in an array.

In this way our algorithm will overcome the unstructuredness of physical claim files. Moreover we used javascript functions to accurately find and validate the policy start and end date.
